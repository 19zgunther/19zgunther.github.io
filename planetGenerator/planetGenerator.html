About this project
I have always been interested in procedurally generated terrain and objects. The idea that I can generate different 3d objects by simply modifying a few input parameters prompted me to build this model.

For this project I learned WebGL as opposed to basic html contexts for increased performace, as I knew I would likely need to render thousands of triangles each frame (30 times per second). WebGL requires all object models to be defined in the vertice-indice model, where vertices are 3d points in space and the indices are triples of vertice indexes defining triangles.

To generate spherical objects (planets and astroids in this project) with varying levels of definition, a simple algorithm is used. Starting with an octohedron, each triangle side is converted into 4 smaller triangles by bisecting the edges of the original triangle. The new vertices are then adjusted to be exactly 1 radius away from the origin. By repeating this process a few times, the octoheron can easily be transformed into a rough sphere made from triangles. Next, the vertices can be pushed and pulled randomly towards and away from the origin, to give the object some more interesting detail. For planet generation, randomly generated rays (50-100 rays) are shot outward from the origin, with vertices within a random distance being pushed farther from the origin. This randomly creates patterns that with the addition of height-dependent coloring produces an earth-like planet object.
